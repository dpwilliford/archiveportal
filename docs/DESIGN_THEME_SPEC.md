# Design and Theme Specification

## Principle

The public portal should resemble a contemporary archival publication platform rather than a generic corporate website.

The design must prioritize:

- readability
- accessibility
- archival context
- public trust
- calm navigation
- visual clarity
- media legibility
- mobile usability

The design must not prioritize:

- engagement optimization
- addictive interaction patterns
- infinite-scroll attention capture
- sensationalized presentation of sensitive material

## Theme goals for Slice 5

The first public launch requires:

- coherent visual identity
- SVG logo in the site header
- responsive layout
- readable typography
- accessible navigation
- minimal but stable design system
- CMS-compatible page layouts

The design does not need to be visually final at Slice 5.

## Header requirements

The header must include:

- SVG logo
- site title or archive title
- primary navigation
- responsive mobile navigation
- visible link to contact page
- accessible focus states

The SVG logo should:

- scale cleanly on mobile and desktop
- render sharply on high-density displays
- support transparent backgrounds
- avoid raster-only dependence

## Typography

Typography must prioritize readability.

Requirements:

- responsive type scale
- readable line length
- adequate line spacing
- semantic heading hierarchy
- distinguishable link styling
- visible focus states

## Color and contrast

The theme must:

- meet accessible contrast expectations
- avoid low-contrast text
- avoid reliance on color alone for meaning
- maintain readable forms and navigation

## Layout system

The design system should include:

- page container widths
- spacing scale
- section spacing
- card layouts
- responsive grid behavior
- footer layout
- form layouts
- alert/banner components

## CMS compatibility

The theme must support:

- Markdown or rich-text content
- long-form essays
- publication pages
- exhibit pages
- media captions
- metadata blocks
- contributor pages

## Homepage expectations for Slice 5

The homepage may initially include:

- hero section
- mission statement
- archive overview
- volunteer/contribute section
- upcoming events placeholder
- publication/exhibit placeholders
- contact call-to-action

## Form design

Forms must:

- remain readable on mobile
- show validation states clearly
- expose errors clearly
- maintain accessible labels
- avoid hidden required information

## Accessibility expectations

The design system must support:

- keyboard navigation
- semantic HTML structure
- screen-reader compatibility
- accessible form labels
- visible focus indicators
- readable mobile layouts

## Explicit material presentation

Sensitive or explicit material must:

- avoid sensationalized presentation
- suppress thumbnails where required
- avoid autoplay behavior
- avoid aggressive visual emphasis
- maintain contextual framing

## Future expansion

Later slices may add:

- exhibit-specific layouts
- timeline visualizations
- advanced media galleries
- oral-history presentation layouts
- digital-book layouts
- community publication layouts
- dark mode if appropriate
